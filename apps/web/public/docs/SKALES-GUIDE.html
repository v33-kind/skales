API Key Setup

For cloud providers, you'll need an API key from your chosen service. Add it in Skales Settings > AI Provider.

Local servers don't require API keys - leave that field empty and Skales will connect via OpenAI Compatible protocol.

KoboldCpp Setup

KoboldCpp brings local AI to your PC with one-click model loading and built-in features.

  1. Download from koboldai.org/cpp
  2. Launch KoboldCpp and select a GGUF model to load
  3. In Skales, go to Settings > AI Provider and choose OpenAI Compatible
  4. Enter: http://127.0.0.1:5001/v1
  5. Leave the API key field blank
  6. Any model name works when router mode is disabled

Ollama Setup

Ollama makes running large language models on your machine simple and fast.

  1. Install from ollama.com
  2. Pull a model: ollama pull llama2 (or any other model)
  3. Skales auto-detects Ollama. No manual configuration needed.
  4. Works fully offline once the model is downloaded

💡 Auto-Detection

Skales automatically finds Ollama running on localhost. Just install, pull a model, and Skales will see it.

Telegram Bot Setup

Use Skales from your phone via Telegram.

  1. Create a new bot with @BotFather on Telegram
  2. Copy your bot token
  3. In Skales, go to Settings > Integrations > Telegram
  4. Paste the token and click Save & Connect
  5. Send /start to your bot
  6. Enter the pairing code shown in Skales

⚠️ Offline Workaround

If Telegram connectivity is offline, check that localhost resolves correctly. IPv6 issues can prevent connection.

Linux (Beta)

Skales is available for Linux via AppImage. Performance is excellent on modern systems.

Installation

  1. Download the AppImage from skales.app
  2. Make it executable: chmod +x Skales-*.AppImage
  3. Run it: ./Skales-*.AppImage

Known Quirks

FAQ

What do I do if the auto-updater has issues?

Manual updates are always available from skales.app. Download the latest version and reinstall.

Why isn't my Telegram bot connecting offline?

Check that localhost resolves correctly on your system. IPv6 configuration issues can prevent connection to local services.

The chat keeps stopping. What's wrong?

Check your API key is valid, your network connection is stable, and your AI provider account has available credits or quota.

Ollama isn't being detected. How do I fix it?

Make sure Ollama is running on localhost:11434. Restart Skales after starting Ollama. Check firewall settings if Skales still doesn't detect it.

Why does KoboldCpp show an auth error?

Ensure you're using the correct endpoint: http://127.0.0.1:5001/v1. Leave the API key field empty for local servers.
